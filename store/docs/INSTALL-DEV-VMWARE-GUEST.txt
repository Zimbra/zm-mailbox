Setup Development Environment in a guest Ubuntu OS running inside VMWare Workstation or Fusion with Perforce and Eclipse running in the host OS (Windows, Mac or Linux)
This allows having a working dev environment and ability to use an IDE on an OS that is not supported by server architecture
================================================================================

1: Install VMWare Workstation (for Windows) or VMWare Fusion (for Mac)  

2. Download a Ubuntu Desktop 12.04 ISO file (ubuntu-12.04.4-desktop-amd64+mac.iso)

3. Create a new virtual machine using the ISO file

4. Turn off the virtual machine, then create a folder named "ubuntuhome" under your home folder. Create folder "p4" under "ubuntuhome" folder.
Go to virtual machine settings and share "ubuntuhome" folder with your  virtual machine with Read/Write access

5. Turn on the virtual machine and install OpenSSH server in your guest OS. To do so open Terminal app on teh guest OS and run:  
$sudo apt-get install openssh-server


6. Link host's shared folder to your Ubuntu home:
$ ln -s /mnt/hgfs/ubuntuhome/p4 $HOME/p4

7. Download perforce visual client for your host OS from perforce.com, install it, connect to p4 server and create a new workspace.

 8. get the latest revision of the code using your Perforce client in your host OS
 
 9. Download Perforce command line client to your guest OS and copy it to $HOME/bin
 
 on your guest OS:
 $mkdir $HOME/bin
 $cp $HOME/Downloads/p4 $HOME/bin/
 $chmod +x $HOME/bin/p4
 
 10. Set up environment variables and perforce access on your guest OS:
 
 $ vi ~/.profile

  export P4PORT=1066
  export P4HOST=p4proxy.eng.zimbra.com
  export P4USER={your p4 username}
  export P4CONFIG=.p4config
  export P4EDITOR=/usr/bin/vi
  export PATH=$PATH:/opt/zimbra/bin:$HOME/bin
  export ZIMBRA_HOSTNAME={your computer name}.local
  alias ssh_p4='ssh -f -N p4'
  
$vi $HOME/.ssh/config

Host *
    User {your fence username}
    IdentityFile ~/.ssh/id_rsa
    ForwardAgent yes
    ServerAliveInterval 30
    ServerAliveCountMax 120
    
Host p4
   Hostname fence-new.zimbra.com
   LocalForward 1066 perforce.zimbra.com:1066

11. Copy your id_rsa identity file to $HOME/.ssh/ folder on your guest OS and set permissions to 600

12. load new environment settings and start SSH tunnel to perforce:

$ source ~/.profile
$ p4_ssh
 
13. Go to step 12 in INSTALL-DEV-UBUNTU12_64.txt and follow the rest of the steps from step 12 onward. 

Once your are finished, you will be able to call ANT targets inside your guest OS and edit the files using an IDE in your host OS
